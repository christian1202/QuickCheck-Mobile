=======================================================================
1. PROJECT OVERVIEW
=======================================================================

Name:         QuickCheck
Type:         Personal Project (not academic)
Developer:    Solo
Platform:     Mobile — iOS & Android
Category:     Church Attendance Management SaaS

QuickCheck is an offline-first mobile attendance management system
built specifically for churches. It is designed to replace manual
Excel-based tracking used by church secretaries. The primary user
is a non-technical church secretary who needs to take attendance
for 50–300 members quickly, reliably, and without internet dependency.

Target benchmark: A secretary should be able to mark attendance
for 150 members in under 5 minutes.

=======================================================================
2. PRIVACY MODEL
=======================================================================

QuickCheck uses a Google Docs-style access model, NOT end-to-end
encryption. Data is stored in plaintext in the database but protected
by Supabase Row-Level Security (RLS) policies.

What the app OWNER (developer) can see:
  - Secretary name
  - What local/branch they belong to

What the app OWNER cannot see:
  - Member names
  - Attendance records
  - Absence reasons
  - Any congregation data

Each local/branch is a completely isolated data silo enforced by RLS.
Secretaries can only access data belonging to their own local/branch.

=======================================================================
3. TECH STACK
=======================================================================

Frontend:           React Native (Expo) — cross-platform iOS & Android
Local Database:     WatermelonDB — offline-first, fast local queries
Backend & Cloud:    Supabase (PostgreSQL + RLS)
File Storage:       Cloudflare R2 — for absence proof photo attachments
Push Notifications: Expo Push Notifications / Firebase Cloud Messaging
No encryption library — privacy is handled entirely by Supabase RLS

=======================================================================
4. ROLES & PERMISSIONS
=======================================================================

ADMIN
  - Manages secretaries
  - Creates and manages local/branch records
  - Can only see: secretary names and their local/branch
  - Cannot see any congregation data

SECRETARY
  - Full access to their local's data only
  - Manages members, events, attendance, absence reports
  - Views dashboard and all reports
  - Cannot access other locals' data

MEMBER
  - Views their own attendance record
  - Submits absence reasons for their own events
  - Receives push notifications for their ministry group events

VIEWER (Church Leadership)
  - Read-only access to reports and dashboard
  - Cannot edit anything

5. FEATURES
=======================================================================

-----------------------------------------------------------------------
5A. MEMBER MANAGEMENT
-----------------------------------------------------------------------
- Full CRUD: add, edit, archive, delete members
- Bulk import via CSV
- Members are scoped to their local/branch only

Member Profile Fields:
  - Full name
  - Photo
  - Contact number
  - Role/position in the church
  - Ministry group (Youth, Choir, Ushers, Elders, etc.)
  - Member since (date joined)
  - Birthday
  - Emergency contact
  - Status: Active | Inactive | On Leave | Transferred

-----------------------------------------------------------------------
5B. EVENT MANAGEMENT
-----------------------------------------------------------------------
- Create events with: name, date, time, location, event type
- Event types: Sunday Service, Prayer Meeting, Special Event,
  General Assembly, etc.
- Target audience: specific ministry group OR whole church
- Recurring events (e.g. every Wednesday prayer meeting)
  using RRULE format for recurrence rules
- Event templates for frequently created event types
- Pre-event view: full expected member list showing
  excused vs. not-yet-marked members before the event starts

-----------------------------------------------------------------------
5C. ATTENDANCE TAKING
-----------------------------------------------------------------------
- Secretary-only manual marking — no QR codes, no self check-in
- Quick Mark mode: fast-scroll list, one tap per member
- Attendance statuses: Present | Late | Absent
- Late tracking: log exactly how late a member arrived
- Mark All Present bulk toggle at top of list
- Real-time absent list: shows who hasn't been marked yet
  during a live event
- Offline-first: works with zero internet, syncs on reconnect
- Optimistic UI: taps register instantly, sync happens in background

-----------------------------------------------------------------------
5D. ADVANCED CALENDAR
-----------------------------------------------------------------------
- Full calendar view — freely navigate forward and backward
- Tap any date (past or future) to:
    - View events on that day
    - Edit any member's attendance status retroactively
    - Add or update absence reports for that date
    - See full present/late/absent/excused breakdown for the day
- Color-coded date indicators:
    - Green  = event exists, attendance fully marked
    - Yellow = event exists, attendance incomplete
    - Grey   = no event on that day
- Jump-to-date shortcut for quick navigation

-----------------------------------------------------------------------
5E. ABSENCE REPORTING
-----------------------------------------------------------------------
- Secretary writes absence reason on behalf of the member
- Member can also submit their own reason through the app
- Secretary can approve or reject member-submitted reasons
- Internal secretary notes (not visible to the member)
- Absence status tags: Excused | Unexcused | No Response | Under Review
- Reason categories: Health | Work | Family | Travel | No Response | Other
- Proof attachment: photo upload stored on Cloudflare R2

-----------------------------------------------------------------------
5F. LIGHT MODE & DARK MODE
-----------------------------------------------------------------------
- Full light mode and dark mode support across all screens
- Follows the device system setting by default
  (auto-switches when the phone switches modes)
- Manual override option in app settings so the secretary can
  force light or dark regardless of system setting
- All charts, dashboards, calendars, and modals must fully
  support both modes — no hardcoded colors anywhere
- Use React Native's useColorScheme() hook as the base,
  with a custom ThemeContext wrapping the entire app
  to handle the manual override

-----------------------------------------------------------------------
5G. MEMBER FULL REPORT
-----------------------------------------------------------------------
Dedicated per-member report page containing:
  - Complete personal information
  - Full attendance history across all events
  - Overall attendance rate percentage
  - Attendance rate broken down by event type
  - All absence records with reasons, status, and attachments
  - Current and historical consecutive absence streaks
  - Ministry group participation breakdown
  - Month-by-month attendance trend graph
  - Secretary notes and internal remarks history
- Exportable as PDF for leadership sharing

-----------------------------------------------------------------------
5H. DASHBOARD (Secretary Home Screen)
-----------------------------------------------------------------------
Top Summary Cards:
  - Total active members
  - Today's event (if any) with quick-launch attendance button
  - Overall attendance rate this month
  - Number of unresolved absence reports

Visual Data Panels:
  - Attendance trend line graph — last 30 / 60 / 90 days
  - Attendance breakdown pie chart — Present vs Late vs Absent
    vs Excused this month
  - Ministry group comparison bar chart — best/worst attendance
    per group
  - At-risk members list — members below attendance threshold,
    surfaced front and center
  - Upcoming birthdays — next 7 days
  - Upcoming events — next 3 events with expected headcount

Quick Action Buttons on Dashboard:
  - Start today's attendance
  - Add new member
  - Create new event
  - View pending absence reports

-----------------------------------------------------------------------
5I. REPORTS & ANALYTICS
-----------------------------------------------------------------------
- Attendance rate per member
- Event summary report with full percentage breakdown
- Attendance trend graphs over time (per member or per group)
- Most missed event types ranking
- At-risk member auto-flagging (configurable threshold, e.g. <70%)
- Export as PDF or CSV
- All reports use pre-computed summaries — load instantly

-----------------------------------------------------------------------
5J. NOTIFICATIONS
-----------------------------------------------------------------------
- Pre-event reminders to members (configurable: 1 day / 1 hour before)
- Members only notified for events tied to their ministry group
- Real-time secretary alert: who hasn't arrived yet during an event
- Escalation alert: member hits X consecutive absences without reason
- Member notified when absence report is approved or rejected
- "You were marked absent" messages sent AFTER service ends, not during

6. DATABASE SCHEMA (Supabase / PostgreSQL)
=======================================================================

TABLE: locals
  id              UUID, Primary Key
  name            String (branch/local name)
  location        String
  created_at      Timestamp

TABLE: users
  id              UUID, Primary Key
  email           String
  role            Enum: admin | secretary | member | viewer
  local_id        UUID, Foreign Key to locals
  created_at      Timestamp

TABLE: members
  id              UUID, Primary Key
  local_id        UUID, Foreign Key to locals
  full_name       String
  photo_url       String (Cloudflare R2)
  contact_number  String
  role_in_church  String
  ministry_group  String
  member_since    Date
  birthday        Date
  emergency_contact String
  status          Enum: active | inactive | on_leave | transferred
  created_at      Timestamp

TABLE: ministry_groups
  id              UUID, Primary Key
  local_id        UUID, Foreign Key to locals
  name            String

TABLE: events
  id              UUID, Primary Key
  local_id        UUID, Foreign Key to locals
  name            String
  event_type      Enum: sunday_service | prayer_meeting |
                        special_event | general_assembly | other
  date            Date
  time            Time
  location        String
  ministry_group_id UUID, Foreign Key to ministry_groups (nullable)
  is_recurring    Boolean
  recurrence_rule String (RRULE format)
  template_id     UUID, Foreign Key to event_templates (nullable)
  created_at      Timestamp

TABLE: event_templates
  id              UUID, Primary Key
  local_id        UUID, Foreign Key to locals
  name            String
  event_type      Enum (same as events)
  default_time    Time
  default_location String

TABLE: attendance_records
  id              UUID, Primary Key
  event_id        UUID, Foreign Key to events
  member_id       UUID, Foreign Key to members
  status          Enum: present | late | absent
  minutes_late    Integer (nullable, only if status = late)
  marked_by       UUID, Foreign Key to users (secretary who marked it)
  marked_at       Timestamp
  updated_at      Timestamp

TABLE: absence_reports
  id              UUID, Primary Key
  attendance_record_id UUID, Foreign Key to attendance_records
  submitted_by    UUID, Foreign Key to users
  reason_category Enum: health | work | family | travel |
                        no_response | other
  reason_text     String
  proof_url       String (Cloudflare R2, nullable)
  status          Enum: excused | unexcused | no_response | under_review
  secretary_note  String (internal, not visible to member)
  reviewed_by     UUID, Foreign Key to users (nullable)
  reviewed_at     Timestamp
  created_at      Timestamp

TABLE: attendance_summary  (pre-computed, updated on every mark)
  id              UUID, Primary Key
  member_id       UUID, Foreign Key to members
  month           Date (first day of month)
  total_events    Integer
  present_count   Integer
  late_count      Integer
  absent_count    Integer
  excused_count   Integer
  attendance_rate Float (percentage)
  updated_at      Timestamp

7. PERFORMANCE NON-NEGOTIABLES
=======================================================================

- Pagination: load members in chunks of 30-50, never all at once
- Local search only: all member search runs against WatermelonDB,
  zero network calls, must feel instant
- Delta sync: only push/pull records changed since last sync,
  never full dataset
- Pre-computed summaries: attendance_summary table updated in
  background on every mark, reports and dashboard read from it
- Background CSV import: encryption/processing runs off main thread,
  UI never freezes during bulk import
- FlatList with getItemLayout: required for Quick Mark mode,
  enables smooth scrolling on mid-range Android phones
- Optimistic UI: attendance status updates immediately on tap,
  Supabase sync happens in the background silently
- Offline-first: all core features (marking, viewing, editing)
  must work with zero internet connection
